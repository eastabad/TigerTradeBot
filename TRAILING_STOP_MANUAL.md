# 智能移动止损系统 (Intelligent Trailing Stop System)

## 系统概述

本系统是一个综合性的智能移动止损解决方案，采用 Tiger Securities API 作为唯一数据源获取实时市场数据和执行交易。系统采用两阶段止损策略：先使用阶梯式止损锁定利润，再根据趋势强度智能切换到动态移动止损，实现对持仓的精准保护。

---

## 完整工作流程

### 第一阶段：开仓与追踪激活

```
TradingView 信号 → /webhook 或 /webhook_paper 接收
                          ↓
              解析信号，获取 timeframe、symbol、side、quantity
                          ↓
              Tiger API 执行开仓 + 设置固定止损/止盈子订单
                          ↓
              创建 TrailingStopPosition 记录
              - 记录入场价、数量、方向
              - 记录固定止损/止盈价格和订单ID
              - 记录信号 timeframe（用于ATR计算）
              - 初始化 highest_price/lowest_price = 入场价
              - 设置 is_active = True
              - 设置 progressive_stop_tier = 0
```

## 外部 Webhook 信号格式

发送至: `https://webhook-migration.replit.app/webhook/signal`

### 1. 取消订单 (Cancel)

当需要取消现有挂单时发送：

```json
{
    "ticker": "AAPL",
    "action": "cancel"
}
```

### 2. 设置止损止盈 (Exit)

阶梯止损上移时（保留原止盈）：

```json
{
    "ticker": "AAPL",
    "action": "exit",
    "stop_loss": {
        "stop_price": 250.50
    },
    "take_profit": {
        "limit_price": 280.00
    }
}
```

切换到移动止损后（仅止损，无止盈）：

```json
{
    "ticker": "AAPL",
    "action": "exit",
    "stop_loss": {
        "stop_price": 255.00
    }
}
```

### 3. 平仓信号 (Close)

```json
{
    "action": "sell",
    "ticker": "AAPL",
    "quantity": "all",
    "sentiment": "flat"
}
```

### Webhook 发送流程

- **阶梯止损上移**: cancel → 等待0.3秒 → exit (新止损 + 原止盈)
- **切换到移动止损**: cancel → 等待0.3秒 → exit (仅止损)
- **触发平仓**: 直接发送 close 信号

## 阶梯止损系统 (Progressive Stop Loss)

### 4层阶梯定义

| 阶梯 | 浮盈触发条件 | 止损移动到 | 配置参数 |
|------|-------------|-----------|---------|
| Tier 1 | ≥ 1% 浮盈 | 盈亏平衡 (0%) | prog_tier1_profit, prog_tier1_stop_at |
| Tier 2 | ≥ 3% 浮盈 | +1% 利润 | prog_tier2_profit, prog_tier2_stop_at |
| Tier 3 | ≥ 5% 浮盈 | +3% 利润 | prog_tier3_profit, prog_tier3_stop_at |
| Tier 4 | ≥ 8% 浮盈 | +5% 利润 | prog_tier4_profit, prog_tier4_stop_at |

### 阶梯切换逻辑

```python
if profit_pct >= 0.08:  # 8%
    target_tier = 4
elif profit_pct >= 0.05:  # 5%
    target_tier = 3
elif profit_pct >= 0.03:  # 3%
    target_tier = 2
elif profit_pct >= 0.01:  # 1%
    target_tier = 1
else:
    target_tier = 0  # 保持原始止损

# 只上调不下调
if target_tier > current_tier:
    execute_stop_adjustment()
```

### 实际案例

**案例：股票 AAPL 入场价 $200**

| 时间点 | 当前价格 | 浮盈% | 阶梯 | 止损价 | 锁定利润 |
|--------|---------|-------|------|--------|---------|
| 开仓 | $200 | 0% | 0 | $190 (原始) | - |
| T+1 | $202 | 1% | 1 | $200 (平衡) | $0 |
| T+2 | $206 | 3% | 2 | $202 (+1%) | $2 |
| T+3 | $210 | 5% | 3 | $206 (+3%) | $6 |
| T+4 | $216 | 8% | 4 | $210 (+5%) | $10 |
| T+5 | 趋势评分>60 | - | 切换移动止损 | 动态计算 | - |

## 趋势强度评分系统

### 评分概述

趋势强度是一个0-100的综合评分，用于判断在达到Tier 4后是否应该切换到动态移动止损。只有当趋势强度 > 60 时才会切换。

### 三个评分指标

#### 1. ATR收敛度评分 (权重30%)

比较当前ATR与近期平均ATR的比值：

| ATR收敛度 | 含义 | 评分 |
|-----------|------|------|
| < 0.70 | 波动大幅收敛，趋势稳定 | 100 |
| 0.70 - 0.85 | 波动明显收敛 | 70 |
| 0.85 - 1.00 | 波动略有收敛 | 50 |
| 1.00 - 1.20 | 波动略有扩大 | 30 |
| > 1.20 | 波动大幅扩大，趋势不稳 | 0 |

#### 2. 动量评分 (权重40%)

计算公式：`动量 = (当前价格 - N根K线前价格) / ATR`

| 动量值 | 含义 | 评分 |
|--------|------|------|
| ≥ 3.0 | 极强动量，价格移动超过3倍ATR | 100 |
| 2.0 - 3.0 | 强动量 | 80 |
| 1.0 - 2.0 | 中等动量 | 60 |
| 0.5 - 1.0 | 弱动量 | 40 |
| 0 - 0.5 | 微弱动量 | 20 |
| < 0 | 反向动量 | 0 |

#### 3. 连续创新高/低评分 (权重30%)

统计最近K线连续创新高（多头）或新低（空头）的次数：

| 连续次数 | 含义 | 评分 |
|----------|------|------|
| ≥ 5 | 极强趋势 | 100 |
| 4 | 很强趋势 | 80 |
| 3 | 强趋势 | 60 |
| 2 | 中等趋势 | 40 |
| 1 | 弱趋势 | 20 |
| 0 | 无趋势 | 0 |

### 综合评分计算

```
趋势强度 = ATR收敛评分 × 0.3 + 动量评分 × 0.4 + 连续创新高评分 × 0.3
```

### 切换条件

| 趋势强度 | 动作 |
|----------|------|
| ≤ 60 | 保持阶梯止损，不切换 |
| > 60 | 切换到动态移动止损 |

配置参数: `trend_strength_threshold` (默认 60)

## 动态百分比移动止损

切换到移动止损后，系统使用动态百分比公式计算止损距离：

### 分段线性公式

| 浮盈区间 | 回撤计算公式 | 说明 |
|---------|-------------|------|
| 0% ~ 2% | 0% + (浮盈/2%) × 0.2% | 从0%增长到0.2% |
| 2% ~ 5% | 0.2% + ((浮盈-2%)/3%) × 0.3% | 从0.2%增长到0.5% |
| > 5% | 0.5% + ((浮盈-5%)/3%) × 0.3% | 从0.5%增长，上限0.8% |

### 设计理念

- 浮盈越高，允许的回撤空间越大
- 给盈利头寸更多的呼吸空间
- 同时确保利润不会大幅回吐

### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| dynamic_pct_tier1_upper | 0.02 (2%) | 第一区间上限 |
| dynamic_pct_tier2_upper | 0.05 (5%) | 第二区间上限 |
| dynamic_pct_tier1_percent | 0.002 (0.2%) | 第一区间回撤 |
| dynamic_pct_tier2_percent | 0.005 (0.5%) | 第二区间回撤 |
| max_percent_stop | 0.008 (0.8%) | 最大回撤上限 |

## ATR 计算规则

### ATR 计算公式

```
True Range = max(
    当日最高价 - 当日最低价,
    |当日最高价 - 昨日收盘价|,
    |当日最低价 - 昨日收盘价|
)

ATR(N) = SUM(True Range, N) / N
```

默认周期 N = 14 (可通过 atr_period 配置)

## 核心架构

### 数据流

```
Tiger API (实时行情 + K线) → ATR计算 → 阶梯止损 / 移动止损逻辑
                                              ↓
                         Tiger API (订单执行) + Discord通知 + 外部Webhook
```

### 四层保护机制

1. **固定止损** - Tiger 子订单，快速执行（第一道防线）
2. **阶梯止损** - 系统每5秒检查，锁定利润（第二道防线）
3. **反向保护** - 价格反向移动时，根据趋势强度收紧止损（第三道防线）
4. **动态移动止损** - 切换后跟踪最高价，紧密保护（第四道防线）

## 反向保护系统 (Inverse Protection)

### 概述

反向保护是一个独立于阶梯止损和动态移动止损的防御机制。当价格向亏损方向移动时，系统根据趋势强度评分来判断当前趋势是否还有效：如果趋势已经变弱，主动收紧止损以减少亏损；如果趋势仍然强劲，则保持原止损不动，给仓位更多空间。

### 触发条件

```
亏损比例 = 当前亏损 / 止损距离

多头: 当前亏损 = 入场价 - 当前价格,  止损距离 = 入场价 - 固定止损价
空头: 当前亏损 = 当前价格 - 入场价,  止损距离 = 固定止损价 - 入场价

触发阈值: 亏损比例 >= 50% (即价格已走过止损距离的一半)
```

### 收紧规则（基于趋势强度）

| 趋势强度 | 含义 | 收紧系数 | 动作 |
|----------|------|----------|------|
| 0 ~ 30 (弱趋势) | 趋势基本消失 | 0.60 | 止损收紧至止损距离的60%，快速止损 |
| 30 ~ 60 (中等趋势) | 趋势减弱 | 0.70 | 止损收紧至止损距离的70% |
| 60+ (强趋势) | 趋势仍然强劲 | 不变 | 保持原止损，不做调整 |

### 止损计算公式

```
多头: 新止损 = 入场价 - (止损距离 × 收紧系数)
空头: 新止损 = 入场价 + (止损距离 × 收紧系数)
```

### 只收紧不放松

反向保护遵循"只收紧"原则：

```
多头: 新止损 > 当前止损 → 更新,  否则不动
空头: 新止损 < 当前止损 → 更新,  否则不动
```

### 与动态移动止损的优先级

每个检查周期中，系统同时计算动态移动止损和反向保护止损，取更紧的值：

```
多头: 最终止损 = max(动态移动止损, 反向保护止损)
空头: 最终止损 = min(动态移动止损, 反向保护止损)
```

### 执行流程

```
┌─────────────────────────────────────────────────────────────┐
│ 每5秒检查周期                                               │
│                                                              │
│   ① 计算动态移动止损 (calculate_trailing_stop)              │
│   ② 计算反向保护止损 (calculate_inverse_protection_stop)    │
│   ③ 取两者中更紧的值作为最终止损                             │
│                                                              │
│   如果反向保护止损更紧:                                      │
│   ├── 查找当前止损订单ID (get_actual_stop_order_id)         │
│   ├── 找到 → 修改止损订单 (modify_stop_with_retry, 最多3次) │
│   ├── 找不到 → 自动创建新止损订单 (create_or_recover_stop)  │
│   ├── 记录 inverse_protection 事件日志                      │
│   └── 发送 Discord 通知                                     │
└─────────────────────────────────────────────────────────────┘
```

### 实际案例

**案例：股票 AAPL 入场价 $200，固定止损 $190 (止损距离 $10)**

| 场景 | 当前价格 | 亏损 | 亏损比例 | 趋势强度 | 结果 |
|------|---------|------|----------|----------|------|
| 价格微跌 | $196 | $4 | 40% | - | 不触发（< 50%阈值） |
| 价格跌至50%位 | $195 | $5 | 50% | 75 (强) | 不收紧（趋势强劲） |
| 价格跌至50%位 | $195 | $5 | 50% | 45 (中) | 止损收紧至 $193 (70%) |
| 价格跌至50%位 | $195 | $5 | 50% | 20 (弱) | 止损收紧至 $194 (60%) |

**收紧计算详解 (弱趋势):**
```
止损距离 = $200 - $190 = $10
收紧系数 = 0.60 (弱趋势)
新止损 = $200 - ($10 × 0.60) = $200 - $6 = $194
原止损 $190 → 收紧至 $194 (减少 $4 风险)
```

### 安全机制

1. **止损订单丢失自动恢复**: 如果查不到止损订单ID，系统自动创建新的止损订单
2. **修改失败重试**: 止损订单修改最多重试3次
3. **直接API修改**: 反向保护通过Tiger API直接修改止损订单，不经过外部webhook
